\hypertarget{md_README_autotoc_md9}{}\doxysubsection{\texorpdfstring{Views}{Views}}\label{md_README_autotoc_md9}
A {\ttfamily \doxylink{classTensor_1_1View}{Tensor\+::\+View}} is a non-\/owning window into an existing tensor\textquotesingle{}s data. It shares the same underlying memory, so reads and writes through a view are immediately visible in the original tensor and vice versa.\hypertarget{md_README_autotoc_md10}{}\doxysubsubsection{\texorpdfstring{Obtaining a view}{Obtaining a view}}\label{md_README_autotoc_md10}

\hypertarget{md_README_autotoc_md11}{}\doxysubsubsection{\texorpdfstring{Assigning through a view}{Assigning through a view}}\label{md_README_autotoc_md11}
Assigning a {\ttfamily \doxylink{classTensor}{Tensor}} (or another {\ttfamily View}) to a view copies data into that window of the original tensor. The original tensor is mutated; the view itself is not reseated.


\begin{DoxyCode}{0}
\DoxyCodeLine{\mbox{\hyperlink{classTensor}{Tensor}}\ a(\{3,\ 4\},\ 0.0f);}
\DoxyCodeLine{}
\DoxyCodeLine{a[0]\ =\ \mbox{\hyperlink{classTensor}{Tensor}}(\{4\},\ 7.0f);\ \ \ \textcolor{comment}{//\ overwrite\ row\ 0\ with\ all\ 7s}}
\DoxyCodeLine{}
\DoxyCodeLine{a[2][3]\ =\ \mbox{\hyperlink{classTensor}{Tensor}}(99.0f);\ \ \ \ \textcolor{comment}{//\ set\ element\ (2,\ 3)\ to\ 99}}
\DoxyCodeLine{}
\DoxyCodeLine{\textcolor{comment}{//\ Copy\ row\ 1\ from\ another\ tensor}}
\DoxyCodeLine{\mbox{\hyperlink{classTensor}{Tensor}}\ src(\{4\},\ 5.0f);}

\hypertarget{md_README_autotoc_md12}{}\doxysubsubsection{\texorpdfstring{Strided views}{Strided views}}\label{md_README_autotoc_md12}
{\ttfamily subsample(strides)} returns a view that steps through the tensor with the given per-\/dimension stride. The view shares the original tensor\textquotesingle{}s memory, so assigning through it writes back directly.


\begin{DoxyCode}{0}
\DoxyCodeLine{\mbox{\hyperlink{classTensor}{Tensor}}\ a(\{6\},\ 0.0f);}
\DoxyCodeLine{\textcolor{keywordflow}{for}\ (\textcolor{keywordtype}{size\_t}\ i\ =\ 0;\ i\ <\ 6;\ i++)\ a[i]\ =\ Tensor(static\_cast<float>(i));}
\DoxyCodeLine{\textcolor{comment}{//\ a\ =\ [0,\ 1,\ 2,\ 3,\ 4,\ 5]}}
\DoxyCodeLine{}
\DoxyCodeLine{\textcolor{keyword}{auto}\ evens\ =\ a.subsample(\{2\});\ \textcolor{comment}{//\ view\ of\ elements\ at\ indices\ 0,\ 2,\ 4}}

\end{DoxyCode}


Because the view is backed by the same memory, assigning to it mutates the original tensor in-\/place\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\mbox{\hyperlink{classTensor}{Tensor}}\ a(\{6\},\ 0.0f);}
\DoxyCodeLine{\textcolor{comment}{//\ a\ =\ [0,\ 0,\ 0,\ 0,\ 0,\ 0]}}
\DoxyCodeLine{}
\DoxyCodeLine{a.subsample(\{2\})\ =\ \mbox{\hyperlink{classTensor}{Tensor}}(\{3\},\ 9.0f);}
\DoxyCodeLine{\textcolor{comment}{//\ a\ =\ [9,\ 0,\ 9,\ 0,\ 9,\ 0],\ only\ the\ even\ indices\ were\ touched}}

\end{DoxyCode}


The same principle applies to higher-\/rank tensors. A stride of {\ttfamily \{2, 1\}} on a matrix skips every other row while keeping all columns\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\mbox{\hyperlink{classTensor}{Tensor}}\ m(\{4,\ 3\},\ 0.0f);}
\DoxyCodeLine{m.subsample(\{2,\ 1\})\ =\ \mbox{\hyperlink{classTensor}{Tensor}}(\{2,\ 3\},\ 1.0f);}
\DoxyCodeLine{\textcolor{comment}{//\ m\ =\ [[1,\ 1,\ 1],}}
\DoxyCodeLine{\textcolor{comment}{//\ \ \ \ \ \ [0,\ 0,\ 0],}}
\DoxyCodeLine{\textcolor{comment}{//\ \ \ \ \ \ [1,\ 1,\ 1],}}
\DoxyCodeLine{\textcolor{comment}{//\ \ \ \ \ \ [0,\ 0,\ 0]]}}

\end{DoxyCode}
